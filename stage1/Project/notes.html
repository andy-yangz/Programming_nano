Lecture 1: The Basics of the web and HTML

Several important concepts about internet

  • Web: Web to connect different computers. Most famous one is WWW (World Wide Web).
  • HTML: Hyper Text Markup Language. Main part of WWW.
  • urls: What we can use it to access website.
  • HTTP: Hyper Text Transfer Protocal. How html file be tranfered and displayed.
  • Web Application: Applications used on web.


How these works?

At first, user should use the browser, a web application, on his computer try to access a web site, which by entering urls in brower. And through web, by HTTP, the server will send us HTML files. So we can look these html files on our own computer through the brower.

What make up HTML files

There are almost four parts compose HTML files. They are:

  • Text Content: Text strings what we see.
  • Mark Up: What a web page looks like.
  • Reference to other documents: Such like images, videos, and audios.
  • Other Pages: We all familiar with this. Through hyperlink we can jump to other pages.
More details.

HTML Elements

HTML files are made by various elements.
And normally elements are made by three parts, opening tag, content, closing tag.
However, some tags can also have attributes. And even there are some tags called void tag without content and closing tag.
There are some examples.

  • Normal: <b> Bold; <em> Emphasis;
  • With Attributes: <a href=url> Anchor
  • Void Tag: <img src=url> This image tag. It's have no closing tag because has no content.
More details.

Inline & Block

In addition to above variety, there are also a feature called Block is important. Block means a part of region divid with other parts. And we can change features of its looklike or something else. However, Inline is elements in a part of region or block. One easy way to discriminate them is to see will it change line, except <br> element. Example:

  • Block: <p>, <span>, <div>
  • Inline: <em> <br>

HTML file detail

  • Doctype: There is a line to declare document type. Normally, HTML.
  • Head: This part will describe the layout of this html file. It can have CSS, Javascript.
  • Body: Main part of a html file. The contents that we want show to other people will be there.

Lecture 2: The structure of HTML

Funny analogy of HTML, CSS, and Javascript

If a webpage file is an house, then we can look HTML as it's walls and fundation. In other words, it forms the structure of webpage. CSS, which decides the style of the webpage, just like how we choose our room color and decorations. Last, Javascript like the controller in our house. It manages the interactions of the webpage.

Chrome developer tool

If we use chrome, we can access some very covenient tool for our development. For shortcut, we can press Shift + Ctrl + i to call developer tool. From this tool, we could see structure of webpage very easily, and we can also change some features by our own to look what will happen.

DOM (Document Object Model)

DOM means tree-like structure of page. Just like in HTML files, we can see many elements are included in other elements, which also are included by other elements. There are many levels of elements, and you can draw a tree by their relations.

Everywhere boxes, why box

At first, why we use boxes to represent elements is that in this way it will be easy to rearrange the style and layout of your webpage. Because the concept of box is so important that we need learn how to boxify a design. Before you type your code, you need draw boxes to the ordered design and get the structure. Then type your code.

Lecture 3: CSS style and HTML structure

CSS (Cascading Style Sheet)

CSS are the codes to control "Style" of HTML elements. It has two parts: the selector part to select elements; the declaration to declare the property and its value.

OH: span & div, class & id

The main difference between span and div two elements is that span is inline element, but div is block element. In othe words, div will take whole width, while span just take the area it spans. So span is used to adjust more dedicted style, whereas div is used to control layout style.

The differece between id and class is that id always be used to a unique element, while class is used to a group of elements. So id are very convenient to be used as bookmarks of content.

Box model

Box model of HTML has four areas. The core one is called content, which has the main contents. Then the second one is called padding, which like shield to protect content and clear area. More outer one is border area, which is boder of box and has the same color with box. The last one is called margin, which decied the space between boxes.

How to make a good boxes layout

Because box has many properties, so it is very hard to get right size of a box. We can resolve this in two ways.

  1. Set size in terms of percentage.
  2. Set box-sizing property to border-box for every elements.
Sometiems, we can also use flex box layout to achieve a good layout.

Code, Test, Refine

For a big picture flow, when we write HTML file, we need do several steps.

  1. Look for natural boxes from design.
  2. Look repeated styles by semantic elements.
  3. Write HTML.
  4. Apply styles (From big to small).
  5. Fix things.
Normally, we will come cross 3 to 5 several times to make a page look good.
